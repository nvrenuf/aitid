## Final UI/System Pass Notes

### 5 system-level improvements to implement

1. Tighten the shared hierarchy across all four routes so severity, title, score, and operational status read first.
2. Standardize badge, chip, metadata, and card rhythm so the product feels like one system instead of adjacent surfaces.
3. Compress top-of-page compositions, especially on `/overview`, to surface live posture faster with less explanatory framing.
4. Quiet secondary metadata and helper copy so operator surfaces scan faster and feel more confident.
5. Refine the map rail and detail-page evidence sections so they read like serious intelligence briefs without theatrical treatment.

### 5 things to remove from the current product

1. Builder-language, self-referential design narration, and implementation-meta phrasing.
2. Defensive reassurance copy that explains scope decisions outside the surface where those limits actually matter.
3. Redundant chips or labels that repeat page meaning without changing a user decision.
4. Overlong helper text around workbench and detail sections when the structure already communicates the point.
5. Inconsistent severity emphasis and noisy secondary metadata that compete with the primary record anatomy.

### 5 things to preserve

1. ThreatParallax route structure and current data model.
2. The current honest world-map direction and conservative geography semantics.
3. The live status framing in the header: `Live feed active · Updated: [real timestamp]`.
4. The dark ThreatParallax palette and restrained intelligence-product tone.
5. The existing product scope boundaries around evidence, geography, and threat modeling.
